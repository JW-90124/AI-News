# PRD：静态站加载性能优化

## 背景

站点没有远程字体和大图，主要成本来自高密度 HTML、未压缩的静态资源与公开 JSON，以及不影响首屏的网络请求过早启动。来源页包含数百行目录；来源动态首屏已经内嵌 48 条卡片，但脚本仍会立即拉取全量 `signals.json`；角色页的第三方行情组件也会在未进入视口时启动。

## 目标

1. 导出阶段压缩 CSS、JavaScript 和公开 JSON，不牺牲源文件可维护性。
2. 把主模块脚本放到 Head 中提前发现，继续依赖 `type=module` 保持非阻塞执行。
3. 把全量来源动态改为交互时按需请求，并延后 GitHub Stars 和第三方行情组件，避免与首屏资源竞争。
4. 降低超长来源目录的离屏布局与绘制成本。
5. 优化“全部地域”下拉的边框、箭头、交互状态和移动端布局，同时保留原生 select 语义。

## 非目标

- 不删减公开事件、来源或证据内容。
- 不引入客户端框架或运行时服务。
- 不增加 Service Worker，避免静态快照更新后出现难以解释的缓存不一致。
- 不把本次资源体积改善描述成未经真实网络测量的固定加载时长。

## 成功口径

- 导出后的 CSS 小于 90 KB、核心脚本小于 28 KB、Timeline JSON 小于 350 KB。
- 核心脚本在 `</head>` 前被发现，并且每页只加载一次。
- 来源动态首屏不请求约 1.8 MB 的全量 JSON；搜索、地域筛选和加载更多时才触发加载。
- 行情组件只在距离视口 360px 内开始加载。
- 地域下拉具备自定义箭头、hover、focus 和主题色适配。
