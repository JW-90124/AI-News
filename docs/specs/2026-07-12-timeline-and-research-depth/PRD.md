# PRD：从事件列表升级为技术认知流

## 1. 当前问题

1. 事件脉络仍是连续卡片列表，跨越两年后缺少年月锚点，用户难以建立阶段感。
2. 详情只在移动端表现为抽屉；桌面端常驻双栏压缩列表宽度，也不符合“先扫读、需要时再展开”的消费方式。
3. GitHub 入口混用“Star on GitHub”和数量，缺少用户熟悉的 GitHub Star 组件结构；本地构建还可能没有数字。
4. 已接入 cs.AI、cs.CL、cs.LG、cs.RO 等论文源，但 `research-eval` 没有事件性加分，具备完整摘要和技术贡献的论文常停在事件门槛之外。
5. 公开事件中的论文密度、技术方法、实验结果、局限和决策含义不足，内容更像产品发布摘要。

## 2. 产品目标

- 事件脉络按“年 → 月 → 事件”呈现，并始终按最近进展倒序。
- 点击任意事件，在桌面和移动端都从右侧/底部打开抽屉；URL、关闭、Esc、焦点恢复和浏览器返回行为可用。
- GitHub 顶部入口使用通用的“GitHub 图标 | Star | 数量”分段组件，数量来自构建期 GitHub API 或 Actions 注入值。
- 修复论文贡献进入 review 候选的事件性门禁，但不允许论文自动越过编辑、证据和发布门禁。
- 增加一组经过人工收敛、明确标注为预印本的高价值研究事件；每条都回答方法、结果、局限、产业含义和下一验证信号。
- 首页增加研究前沿摘要，事件脉络增加“论文与研究”筛选，让论文内容成为认知产品的一等入口。

## 3. 事件脉络交互

```text
2026
  7 月 · 近期 12 个事件
    [事件摘要] [事件摘要]
  6 月 · 4 个事件
2025
  12 月 · 3 个事件
```

- 年份只在发生切换时显示；月份标题展示该月事件数。
- 卡片显示最近进展时间、事件类型、主体、事实摘要、进展数和关键字。
- 点击卡片打开抽屉；“打开完整事件”继续进入永久链接。
- 抽屉关闭后焦点回到触发卡片；URL 中存在 `?event=` 时页面加载后直接打开。
- 筛选后没有可见事件的月份和年份自动隐藏。

## 4. 论文内容标准

论文不是因为“新”就值得公开。进入候选必须同时满足：

- Tier 1 research 来源；
- 摘要具备足够长度；
- 标题或摘要明确包含 benchmark、dataset、method、framework、mechanism、architecture 或 evaluation 等可验证贡献；
- 与 LLM、Agent、推理、长上下文、编码、多模态、训练、推理基础设施、对齐或机器人等核心主线相关。

公开前必须补齐：

1. 研究问题；
2. 方法与实验设置；
3. 关键结果；
4. 结论边界与局限；
5. 对技术路线、产品或决策的意义；
6. 下一项可证伪信号。

论文页面必须明确标注“预印本，尚不等于独立复现或同行评审结论”。

## 5. 首批研究内容

使用 2026-07-09 提交至 arXiv cs.AI 的一手预印本建立首批研究事件：

- PredicateLongBench：长上下文难度轴；
- Compete Then Collaborate：可验证课程与编码模型训练；
- AutoPersonas：长期 Agent 自锁与多时间尺度记忆；
- Blind-Spots-Bench：多模态模型盲点；
- Overthinking：推理权重放大与隐藏信息审计；
- CausalDS：数据科学 Agent 的因果推理评测。

## 6. 非目标

- 不把 arXiv firehose 直接发布到公开站；
- 不把预印本表述为已经证实的行业结论；
- 不运行时请求 GitHub API；公开站仍保持静态；
- 不增加新的公开路由；
- 不取消 Event readiness，不降低一手证据、主线、置信度和占位内容门槛。

## 7. 论文可见性补充

研究内容分为两个展示层级，不再把“进入时间轴”和“进入首页精选”使用同一套高门槛：

- **研究时间轴**：所有已经成为公开 Event 的 `research` / `paper` 内容都可通过“论文与研究”筛选查看；允许分析较短，但必须有来源、事实、主线和明确研究贡献。
- **首页研究精选**：继续要求一手证据以及完整的方法、行业影响和下一验证，只展示少量最值得投入时间的论文。

研究候选的摘要长度门槛从 240 字符调整为 160 字符；公开研究分析的最小深度同步适度下调，但占位内容、无一手证据、无主线和低置信度门禁保持不变。

同一 UTC 日期达到 4 个及以上研究 Event 时，在月份列表中聚合为“论文日报”：默认展示日期、数量和主题摘要，展开后保留每一个独立 Event，不合并事实或证据。选择“论文与研究”筛选或搜索命中时自动展开。

事件脉络页主标题缩短为“一个事件，一条完整脉络”，桌面优先保持单行；窄屏通过响应式字号尽量保持单行，不允许造成横向滚动。

## 8. 全站证据抽屉补充

当前论文筛选存在一个显示缺陷：脚本为无匹配内容的月份设置 `hidden`，但 `.timeline-month { display: grid }` 覆盖了浏览器默认的 `[hidden] { display: none }`，导致 2026 年 5 月等月份只剩“1 个事件”标题而没有卡片。必须为月份、年份、聚合组和事件卡显式定义 hidden 样式。

证据抽屉从 Timeline 页面专属能力升级为全站通用阅读能力：

- 首页今日事件、今日还需知道、研究精选；
- 趋势页证据脊柱；
- Timeline 事件卡；
- 事件页相关事件；
- 行动参考中的触发证据；

点击以上入口时默认在当前页面打开证据抽屉，不跳转。抽屉内保留“打开完整事件”作为显式深读动作，并支持 Ctrl/Cmd 点击、鼠标中键和无 JavaScript 环境继续使用原链接。

抽屉不得在每个页面预渲染全部事件详情。页面只输出一个空壳，首次打开时按需读取静态公开 Timeline DTO，之后在当前页面内缓存。加载中、失败、事件不存在都需要明确状态。
