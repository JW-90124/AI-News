# 验收方案

## 1. 产品语言

- 主导航不出现“六条主线、证据时间轴、决策工具”；
- 首页首屏包含明确产品承诺、30 秒 / 3 分钟 / 10 分钟消费路径和今日核心判断；
- CTA 描述用户所得，而不是“打开模块”或“进入工具”。

## 2. 首页

- 1440×900 首屏能看见产品承诺、核心事件标题、事实和至少一个判断维度；
- 390×844 无横向滚动，主 CTA 至少 44px；
- 无公开事件时解释为什么没有内容，并提供查看趋势和来源状态的下一动作。

## 3. 事件脉络

- 事件按最近发展时间倒序；
- 一个 Event 只出现一个列表卡；
- Evidence 在事件内按时间正序，角色文案正确；
- 事件详情包含“发生了什么、改变了什么、为什么重要、对谁有影响、当前判断、接下来观察”；
- URL 中的 event / track 筛选可刷新、返回和复制。

## 4. 覆盖体检

- 14 个要求领域均有覆盖卡；
- Claude Code 能识别 release 来源，且不会因单一健康 release 流显示为完全覆盖；
- Lovable、A2A 若无来源或未检查，明确显示 gap / unchecked；
- 失败来源显示错误分类，不暴露内部错误摘要和样本；
- 来源列表仍能搜索、筛选和打开官方入口。

## 5. 工程质量

- 单元测试覆盖事件排序、发展序列和技术覆盖状态；
- 集成测试覆盖中文/英文页面标题、核心文案、静态链接和隐私扫描；
- `npm run check` 全量通过；
- 导出后检查所有 HTML 内链、canonical、无 `__PREFIX__`、无本机路径；
- 浏览器 smoke 覆盖首页、事件脉络、事件详情、趋势判断、行动参考和来源覆盖。
