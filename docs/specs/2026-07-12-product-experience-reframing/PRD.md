# PRD：从信息展示到认知形成

## 1. 问题定义

当前站点已经具备多页、证据回链、主线、来源和工具，但用户仍需理解“主线、时间轴、证据、系统水位”等内部概念，才能开始消费内容。首页像模块目录，时间轴像事件列表，来源页只能证明目录规模，不能回答具体技术领域是否持续有效覆盖。

这导致三个核心问题：

1. 新用户不能在几秒内理解“为什么每天打开”；
2. 用户知道发生了什么，却不一定完成“理解变化、判断影响、确定观察点”；
3. 系统把“已经收录”和“正在有效观测”混在一起，技术覆盖缺口不可见。

## 2. 产品承诺

> 每天用 10 分钟，看懂 AI 行业真正重要的变化，并形成自己的判断。

首页必须同时回答：

- 30 秒：今天最重要的变化是什么；
- 3 分钟：为什么重要、影响谁、改变了什么；
- 10 分钟：事情如何发展、行业趋势是什么、接下来观察什么。

## 3. 用户消费路径

```text
今日最重要的变化
  -> 一句话事实与判断
  -> 为什么重要 / 影响谁 / 下一观察
  -> 同一事件的发展脉络
  -> 正在形成的行业趋势
  -> 原始来源与深入阅读
  -> 可采取的最小行动
```

## 4. 信息架构

不增加路由，重定义现有入口的用户任务：

| 现有路由 | 用户语言 | 用户任务 |
| --- | --- | --- |
| `/` | 今日必读 | 用最短时间完成今日认知更新 |
| `/lines/` | 趋势判断 | 理解行业变化正在向哪里发展 |
| `/timeline/` | 事件脉络 | 按事件查看从首次出现到当前判断的全过程 |
| `/scout/` | 行动参考 | 把变化转成可验证的创业、产品、内容或工作动作 |
| `/actors/` | 关键参与者 | 理解谁在推动变化以及其证据状态 |
| `/resources/` | 选型与成本 | 获取可核验的模型/API 购买信息 |
| `/product/` | 我们如何判断 | 查看方法、能力边界、样本不足与评测惩罚 |
| `/sources/` | 覆盖与来源 | 检查具体技术领域覆盖、来源健康和盲区 |
| `/changelog/` | 产品更新 | 理解 Agent Pulse 最近改善了什么用户价值 |

导航只突出“今日必读、趋势判断、事件脉络、行动参考”，其余作为深入入口。

## 5. 首页结构

1. 产品承诺与阅读路径：一句话说明价值，标明 30 秒 / 3 分钟 / 10 分钟。
2. 今日核心判断：最重要事件的事实、变化、影响和下一观察。
3. 今日还需知道：近期高价值事件的快速扫读，不重复同一事件。
4. 正在形成的趋势：展示当前判断、最近变化和下一验证信号。
5. 形成判断：引导进入事件脉络、来源核验和行动参考。
6. 深入阅读：原始证据、角色、成本信息按用户任务表达。
7. 方法与品牌说明移到页面末端。

## 6. 事件脉络

- 默认按事件最近一次发展时间倒序，而不是按 featured 或新闻条目排序；
- 一个卡片代表一个 Event，Event 内的 Evidence 是发展节点；
- 发展节点按时间正序展示，并根据证据角色表达为“首次出现、官方更新、外部讨论、行业反馈”；
- 最后追加“当前判断”，明确它是 Agent Pulse 分析而非外部事实；
- 无足够证据时不制造虚假阶段；
- 事件详情与事件脉络页使用同一发展序列。

## 7. 内容表达

事件内容统一为：

```text
发生了什么（事实）
  -> 改变了什么（背景和变化）
  -> 为什么重要（行业影响）
  -> 对谁有影响（决策价值）
  -> 当前怎么看（有证据边界的判断）
  -> 接下来观察什么（可证伪信号）
```

减少 Timeline、Evidence、Decision Tools、Capability Map 等内部语言。必要的证据术语只用于解释可信度，不作为主要导航语言。

## 8. 技术覆盖体检

来源页新增覆盖领域：Claude Code、OpenAI/Codex、Google DeepMind、Cursor、Windsurf、Lovable、Vercel AI、Cloudflare AI、MCP、A2A、Browser Use、AI Coding、AI Infra、AI Agent。

每个领域展示：

- 匹配来源数；
- 最近健康来源数；
- active / observing 状态；
- 已覆盖的渠道类型；
- 明确缺口和下一步补强方向。

不得把“目录中存在”描述成“有效覆盖”。没有最近检查时显示“待验证”；失败或只有单一 release 流时显示“需补强”。

## 9. 非目标

- 不新增新页面或改成 SPA；
- 不自动生成未经审核的事件分析；
- 不把来源目录数量包装成覆盖质量；
- 不用手工热度代替真实传播证据；
- 不在本轮改变 Event、Track 或 Source 生命周期 schema。
