# PRD：补齐重点厂商的事件与证据覆盖

## 背景

当前 Source Catalog 已登记许多模型厂商，但公开 Timeline 并没有形成对称覆盖。智谱、Grok 只是最先暴露的问题；审计还发现 Cohere、字节 Seed、腾讯混元、百度文心、阶跃星辰、零一万物、百川、面壁、商汤、讯飞与 Perplexity 等厂商存在“有来源、无 Event”的断层。搜索只索引 Event 的标题、公司、事实与关键词，因此来源登记本身无法让用户在事件脉络中检索到这些厂商。

根因是系统缺少“重点厂商 -> 官方来源 -> 可核验 Event -> 搜索别名”的覆盖约束。只按用户举例补两个品牌，会继续让其他厂商以同样方式漏出。

## 目标

1. 建立 22 家重点厂商覆盖矩阵，涵盖全球前沿模型、中国重点模型团队与 AI 原生搜索产品。
2. 新增 27 个一手来源入口，覆盖官方发布记录、模型文档和可持续更新的官方 GitHub Atom Feed。
3. 新增 32 个经过人工核验的关键 Event：智谱 / Z.ai 与 xAI / Grok 18 个，其他重点厂商 14 个。
4. 每个 Event 明确区分事实、行业判断、技术变化、业务影响与下一观察，并回链官方页面。
5. Timeline 可检索中英文厂商名、模型品牌和常用产品别名。
6. 新来源保持 `shadow`，目录登记和一次可访问探测不等于投产或独立证据数量。
7. 通过自动测试阻止重点厂商重新退化为“只有 Source、没有 Event”。

## 非目标

- 不绕过 xAI 新闻页的 Cloudflare / WAF。
- 不把第三方媒体、聚合站或社交转述作为重大事实的唯一证据。
- 不把 GitHub commit 自动升级为公开 Event。
- 不因补齐历史事实而改变现有 Event readiness 或自动发布门禁。

## 成功口径

- 27 个新增来源 slug、endpoint 唯一，全部 `disabled + shadow`。
- 17 个 RSS/Atom 端点与 10 个官方 HTML 页面进入统一 SourceAdapter 契约。
- Timeline 新增 32 个关键节点，并覆盖矩阵中的 22 家重点厂商。
- 中文和英文静态页面均包含对应事件，前端搜索可匹配中英文厂商、模型和产品别名。
- 自动测试验证每家重点厂商至少关联两个官方来源入口，并至少有一个公开 Event。
- 所有 Event 使用 Tier 1 官方来源，heat 保持 0，不把人工分数包装为传播热度。
