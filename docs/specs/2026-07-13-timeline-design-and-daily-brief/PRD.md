# PRD：让趋势、事件与行动形成完整认知路径

## 1. 当前问题

1. Timeline 顶部的“论文批次状态”和“近三月密度”属于系统运营信息，占用了用户理解事件的首屏空间。
2. 现有六条主线中“技术演进”和“AGI 进展”高度重叠；“模型经济学”过窄；“中国追赶”把多元创新错误表达为单向落后关系。
3. 趋势卡片虽然整体可点击，但缺少明确按钮；阶段卡片彼此割裂，不能一眼看出阶段传导关系。
4. 事件抽屉宽度、信息层级和 Timeline 行布局不足，事实、背景、技术变化、影响、判断、下一观察没有完整呈现。
5. Timeline 筛选仍使用“一手证据”等系统术语，用户不知道筛选后获得什么。
6. 行动参考存在同一事件、同一类型的重复卡片；标签右对齐且在窄宽度被截断；公开页没有形成足够的信息密度。
7. 当前公开历史从 2024 年开始，缺少 2022 年生成式 AI 产品化起点与 2023 年模型、Copilot、开放生态和治理扩散。
8. 产品只展示成功节点，没有表达创业项目被收购、停止独立运营、关闭产品或转向其他商业路径。
9. 产品把“每日采集”错误地表达成“每日都有新认知”，并用固定 10 分钟承诺和日报制造不真实的消费预期。

## 2. 趋势判断重组

保留六个一级方向，但按不同决策问题重新定义：

| 方向 | 回答的问题 | 由现有维度演化 |
| --- | --- | --- |
| 模型能力与研究 | 模型和研究范式发生了什么实质变化？ | 技术演进收敛模型/研究事实 |
| Agent 与软件重构 | AI 如何从回答问题进入执行任务和重写软件？ | AGI 进展改为可观察的软件行动 |
| 产品与商业验证 | 哪些产品形成采用、留存、收入和工作流控制点？ | 商业化扩展产品验证 |
| 基础设施与成本 | 算力、训练、推理、开源和单位任务成本如何变化？ | 模型经济学扩展为完整供给层 |
| 资本与公司演化 | 资金、并购、团队迁移和创业项目生死意味着什么？ | 投资视角加入公司生命周期 |
| 全球创新版图 | 中国、美国、欧洲与开放生态各自创造了什么？ | 替代“中国追赶”，不预设单向领先关系 |

“中国进展与全球坐标”统一改为“全球创新中的中国实践”，只做同维度事实比较，不使用追赶叙事。

## 3. 2022—今天的行业弧线

公开 Timeline 至少从 2022 年 8 月 Stable Diffusion 和 2022 年 11 月 ChatGPT 开始，形成以下阶段：

1. 2022 H2：生成式 AI 从实验室进入公众产品；
2. 2023 H1：GPT-4、Copilot、插件和企业入口扩散；
3. 2023 H2：开放模型、多模态与全球治理框架成形；
4. 2024：原生多模态、长上下文、推理时计算和早期 Agent；
5. 2025：Agent 协议、编码 Agent、低成本推理与产业重组；
6. 2026：长任务、真实工作、科学应用与平台集中。

每个阶段展示代表模型、产品、创业项目、基础设施、治理变化和公司生命周期。项目状态只允许使用官方公告或仍可核验的产品状态，区分 `active`、`pivoted`、`acquired`、`sunset`，不把“热度下降”伪装成公司死亡。

## 4. 页面体验

- Timeline 首屏直接进入搜索和用户价值筛选，不显示运营密度或论文批次模块。
- 筛选改为“全部变化”“官方发布”“论文研究”及六条趋势；“官方发布”明确表示只看公司、实验室或监管机构的原始公告。
- 趋势目录卡和首页趋势卡都提供可见的“查看趋势详情”按钮。
- 阶段轨迹使用箭头连接，桌面横向、移动端纵向，展示阶段之间的演进关系。
- 抽屉桌面宽度提升到 720–780px，分为事实摘要、事件发展、变化解读、决策影响、下一观察、原始证据和完整页面入口。
- 抽屉 Timeline 不再压缩成两列，日期、节点类型、标题、来源和摘要各自拥有稳定位置；默认展示全部去重证据。
- 统一 reading journey、section、Timeline 底部、抽屉和行动参考的宽度与垂直间距。
- 行动参考按归一化标题与证据事件去重；标签左对齐、允许换行；卡片显示目标用户、置信度、证据分、新颖度、为什么现在、最小动作、产物和反向信号。

## 5. 事件驱动判断与每周 GitHub Issue

- Data Refresh 继续承担高频监测，不把运行次数、抓取数量或快照更新时间包装成趋势变化。
- 趋势判断只在出现以下至少一类公开证据时更新：具有真实传播证据的热点、改变能力/价格/工作流的产品更新、具有可复现实质增量的技术迭代，或同等级的政策、资本与公司转折。
- 没有新证据时保留原判断和原更新时间，页面明确表达“当前判断未变”，不生成占位认知。
- 取消每日 GitHub Issue、`daily-brief` 标签入口和日报 renderer。
- 每周按 Asia/Shanghai ISO 周生成一个 `weekly-brief` Issue；重复运行更新同一 marker，不重复创建。
- 周报只消费公开 allowlist DTO，区分“本周改变判断的事件”“新增证据但判断未变”“持续观察”，不暴露数据库或原始 payload。

## 6. 验收

- Timeline 页面不再包含两个运营模块。
- 全站不出现“中国追赶”或 `China catch-up` 的用户可见文案。
- 六个趋势维度、按钮和详情页一致；阶段轨迹有箭头关系。
- 公开 Timeline 覆盖 2022、2023、2024、2025、2026。
- 至少展示 5 个有官方证据的项目转型、收购或终止节点。
- 抽屉信息完整、桌面和 390px 均无溢出；底部留白与全站 spacing token 一致。
- 星探去重后无重复标题，所有通过公开门禁的高置信内容均可见。
- Data Refresh 每天抓取并刷新站点，只在周日最终窗口或手动触发时创建/更新周报；全仓不再存在日报命令和自动日报 Issue 流程。
- Pages 仅在去除生成时间等波动字段后的公开内容指纹发生变化时部署；纯运行记录增长不触发站点重建。

## 追加需求：首屏层级与星探供给（v0.8.1，阅读时长承诺已退役）

- 首页保留紧凑的三段消费路径，但不再承诺 30 秒、3 分钟、10 分钟或每天打开；改为关键变化、证据脉络、每周复盘。
- 趋势总览用六个趋势入口替换“公开事件数 / 时间跨度 / 选择主线”状态标签，直接放在标题说明下方。
- 星探卡片收紧标题与正文间距，不牺牲标签、证据和行动信息。
- 星探公开供给覆盖创业、内容、工作、学习、产物、影响力六类，自动维持至少 18 条通过既有质量门禁且互不重复的公开机会。
- 达到公开池水位后停止无界生成；过期自动生成卡归档后再补位，seed 长期基线不因无过期时间被清理。

## 追加需求：高密度趋势判断与 GitHub 订阅（Unreleased）

### 问题

1. 六条趋势都被压成三个阶段，阶段不是从真实事件演进中得出，无法表达 2022—今天不同主线各自的节奏。
2. 趋势详情只展示最多 7 条官方事件；既有 266 个来源、191 个事件和 2,633 条信号没有按阶段组织为可消费证据。
3. 证据节点是同构列表，没有说明每个阶段为什么成立、哪些证据改变了判断、还缺什么反向证据。
4. 四种角色各取一条 Event 字段作为答案，缺少决策结论、影响链、建议动作、失效条件和可回溯证据。
5. 高密度内容没有简洁/全量切换；GitHub Star 只能表达支持，不能形成内容订阅。

### 产品目标

- 每条趋势按真实演进拆成 5—8 个不等阶段；轨迹始终单行横向排列，并允许键盘和触摸横向滚动。
- 取消“最多 7 条”证据上限。页面默认保留每阶段最关键的 2 个节点，用户在对应阶段模块内展开全部公开 Event。
- 每个阶段同时展示阶段判断、转折解读、证据节点数、独立来源数、代表事实和下一验证信号。
- 趋势页明确分开“已形成事实证据”和“持续观察源池”；Source Catalog 不得被包装成已验证证据。
- 四种角色使用人工审阅的决策镜头：核心回答、影响链、建议动作、继续观察和失效条件，并回链相关 Event。
- 页面不再提供全局“简洁 / 全量”切换；默认使用简洁呈现，阶段轨迹、阶段证据、角色判断和观察源池分别在模块内展开，展开后仍不输出 review、raw payload 或管理字段。
- 全站增加 GitHub Watch 订阅入口，并说明用户可只订阅 Issues；公开自动内容只保留每周一次的周报 Issue。

### 数据密度口径

- “3 倍信源”以旧版单页最多 7 个证据入口为基线：趋势页应展示全部实际证据来源，并同时给出至少 21 个相关观察源入口；不足时必须显示缺口，不能补假来源。
- “5 倍数据”以旧版 7 个事件行、每行单层摘要为基线：完整模块展开后目标为每条趋势至少 35 个阶段化信息单元；信息单元包括 Event、阶段解读、来源、角色影响和验证信号。未达到目标的主线显示覆盖差距。
- 数量目标不能降低 Event readiness、来源独立性、许可或发布门禁；已有高质量快照应先被充分消费，再扩大采集。

### 订阅体验

- “Watch 周期更新”直接进入 GitHub 仓库订阅设置，文案提示选择 Custom → Issues，避免把所有代码活动都推给非开发用户。
- 移除日报入口；周报入口指向 `weekly-brief` Issue 列表。
- 周报按 Asia/Shanghai 的 ISO 周聚合，只读取公开 Timeline、Scout 和 Product DTO；每周一个 marker，重复运行更新同一个 Issue。

## 追加需求：全站文案与发布节奏收敛（2026-07-13）

### 产品承诺

> 持续监测 AI 行业，只在重要证据出现时更新判断。

- 首页、导航、SEO、README、订阅区、产品 Changelog 不再使用“每天形成判断”“今日必读”“10 分钟形成判断”等固定频率承诺。
- “最新变化”表示最近一次达到公开与重要性门槛的变化，不暗示当天一定有内容。
- 趋势页继续展示当前判断、证据阶段与下一验证信号；证据没有改变时，判断保持稳定。

### 自动化分层

```text
高频采集与审计           可以按小时运行
  -> 公开 Event 门禁      只有合格事实进入公开层
  -> 公开内容指纹         忽略 generatedAt 等波动字段
       changed            提交快照后部署 Pages
       unchanged          只保存运行证据，不部署 Pages
  -> 周日最终窗口         幂等更新本周 weekly-brief Issue
```

周报可以在没有趋势转折的一周发布，但必须明确说明“判断未变”和仍在等待的验证信号，不能为了周更制造趋势。
